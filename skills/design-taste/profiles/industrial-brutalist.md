# SKILL：工业粗野主义与战术遥测 UI

## 1. Skill 元信息

**名称：** 工业粗野主义与战术遥测界面工程
**描述：** 精通架构融合了世纪中期瑞士排版设计、工业制造手册和复古未来主义航空航天/军事终端界面的 Web 界面。这门学科要求绝对掌握刚性模块化网格、极端排版比例对比、纯粹功利主义调色板，以及模拟模拟退化（半色调、CRT 扫描线、位图抖动）的程序化实现。目标是构建投射原始功能性、机械精度和高数据密度的数字环境，刻意摒弃传统消费 UI 模式。

## 2. 视觉原型

设计系统通过融合两个截然不同但高度兼容的视觉范式来运作。**每个项目选择 ONE 并坚持使用。不要在同一个界面内交替或混合两种模式。**

### 2.1 瑞士工业印刷

源自 1960 年代的企业标识系统和重型机械蓝图。
*   **特征：** 高对比度亮色模式（新闻纸/米白基底）。依赖整体式、重型无衬线排版。由可见分割线勾勒出的严格结构网格。激进、不对称的负空间运用，穿插着超大尺寸、溢出视口的数字或字母。大量使用原色红作为警报/强调色。

### 2.2 战术遥测与 CRT 终端

源自机密军事数据库、遗留大型机和航空航天平视显示器（HUD）。
*   **特征：** 暗色模式独占。高密度表格数据展示。等宽排版的绝对主导。技术框架装置（ASCII 括号、十字准星）的整合。模拟硬件限制（磷光发光、扫描线、低位深度渲染）的应用。

## 3. 排版架构

排版是主要的结构和装饰基础设施。图像是次要的。系统要求在比例、重量和间距上具有极端变化。

### 3.1 宏观排版（结构标题）

*   **分类：** 新怪诞 / 重型无衬线。
*   **最佳 Web 字体：** Neue Haas Grotesk（Black）、Inter（Extra Bold/Black）、Archivo Black、Roboto Flex（Heavy）、Monument Extended。
*   **实现参数：**
    *   **比例：** 使用流式排版以超大比例部署（例如 `clamp(4rem, 10vw, 15rem)`）。
    *   **字距（Letter-spacing）：** 极度紧密，通常为负值（`-0.03em` 到 `-0.06em`），迫使字形形成坚实的建筑块。
    *   **行高（Line-height）：** 高度压缩（`0.85` 到 `0.95`）。
    *   **大小写：** 结构冲击效果独占全大写。

### 3.2 微观排版（数据与遥测）

*   **分类：** 等宽 / 技术无衬线。
*   **最佳 Web 字体：** JetBrains Mono、IBM Plex Mono、Space Mono、VT323、Courier Prime。
*   **实现参数：**
    *   **比例：** 固定且小（`10px` 到 `14px` / `0.7rem` 到 `0.875rem`）。
    *   **字距：** 宽松（`0.05em` 到 `0.1em`）以模拟机械打字机间距或终端矩阵。
    *   **行高：** 标准到紧密（`1.2` 到 `1.4`）。
    *   **大小写：** 独占全大写。用于所有元数据、导航、单位 ID 和坐标。

### 3.3 纹理对比（艺术性破坏）

*   **分类：** 高对比度衬线。
*   **最佳 Web 字体：** Playfair Display、EB Garamond、Times New Roman。
*   **实现参数：** 使用极其克制。必须经过重度后处理（半色调滤镜、1 位抖动）以降解矢量完美性，并在干净的无衬线字体之间创造纹理并置。

## 4. 色彩系统

色彩架构是不妥协的。渐变、柔和投影和现代半透明严格禁止。颜色模拟物理介质或原始发射显示器。

**关键：每个项目选择 ONE 基底调色板并一致使用。绝不在同一界面内混合亮色和暗色基底。**

### 如果瑞士工业印刷（亮色）：
*   **背景：** `#F4F4F0` 或 `#EAE8E3`（哑光、未漂白的文档纸）。
*   **前景：** `#050505` 到 `#111111`（碳墨）。
*   **强调色：** `#E61919` 或 `#FF2A2A`（航空/危险红）。这是唯一的强调色。用于删除线、粗结构分割线或重要数据高亮。

### 如果战术遥测（暗色）：
*   **背景：** `#0A0A0A` 或 `#121212`（停用 CRT。避免纯 `#000000`）。
*   **前景：** `#EAEAEA`（白色磷光）。这是主要文字颜色。
*   **强调色：** `#E61919` 或 `#FF2A2A`（航空/危险红）。相同的红色，相同的规则。
*   **终端绿（`#4AF626`）：** 可选。仅用于一个特定的 UI 元素（例如一个状态指示器或一个数据读数）——绝不用作通用文字颜色。如果不服务于明确目的，完全省略。

## 5. 布局与空间工程

布局必须看起来是数学工程化的。它拒绝传统的 Web 内边距，转而采用可见的隔间化。

*   **蓝图网格：** 严格遵守 CSS Grid 架构。元素不浮动；它们被精确锚定到网格轨道和交叉点。
*   **可见隔间化：** 广泛使用实线边框（`1px` 或 `2px solid`）来划分不同的信息区域。水平规则（`<hr>`）经常跨越整个容器宽度以分隔操作单元。
*   **双峰密度：** 布局在极端数据密度（紧密排列的等宽元数据聚集在一起）和广阔的计算负空间（框定宏观排版）之间振荡。
*   **几何：** 绝对拒绝 `border-radius`。所有角必须恰好为 90 度以强制执行机械刚性。

## 6. UI 组件与符号学

标准 Web UI 约定被替换为功利性、工业化的图形元素。

*   **语法装饰：** 使用 ASCII 字符框定数据点。
    *   *框架：* `[ DELIVERY SYSTEMS ]`、`< RE-IND >`
    *   *方向：* `>>>`、`///`、`\\\\`
*   **工业标记：** 突出整合注册（`®`）、版权（`©`）和商标（`™`）符号，作为结构几何元素而非法律文本。
*   **技术资产：** 整合网格交叉点的十字准星（`+`）、重复垂直线（条形码）、粗水平警告条纹和随机字符串数据（例如 `REV 2.6`、`UNIT / D-01`）以模拟活跃的机械过程。

## 7. 纹理与后处理效果

为防止设计显得纯数字化，通过 CSS 和 SVG 滤镜在前端工程化模拟模拟退化。

*   **半色调与 1 位抖动：** 将连续色调图像或大型衬线排版转换为点阵图案。通过预处理或 CSS `mix-blend-mode: multiply` 覆盖层结合 SVG 径向点图案实现。
*   **CRT 扫描线：** 对于终端界面，在背景上应用 `repeating-linear-gradient` 以模拟水平电子束扫描（例如 `repeating-linear-gradient(0deg, transparent, transparent 2px, rgba(0,0,0,0.1) 2px, rgba(0,0,0,0.1) 4px)`）。
*   **机械噪点：** 一个全局的、低透明度的 SVG 静态/噪点滤镜应用于 DOM 根节点，以在暗色和亮色模式下引入统一的物理颗粒感。

## 8. Web 工程指令

1.  **网格确定性：** 使用 `display: grid; gap: 1px;` 配合对比鲜明的父/子背景颜色，以生成数学上完美、剃刀般细的分割线，无需复杂的边框声明。
2.  **语义刚性：** 使用精确的语义标签构建 DOM（`<data>`、`<samp>`、`<kbd>`、`<output>`、`<dl>`）以准确反映遥测的技术性质。
3.  **排版夹紧：** 仅在宏观排版上实现 CSS `clamp()` 函数，以确保巨大的文本比例在积极缩放的同时，跨视口保持结构完整性。
