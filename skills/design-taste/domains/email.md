# 邮件模板设计规则

> 邮件是最受限的前端环境。所有现代 CSS 技术在此失效。尊重约束，在限制内做出有品味的设计。

---

## 1. 邮件 HTML 技术约束

邮件客户端是前端的时间胶囊。2026 年的 Outlook 仍然用 Word 引擎渲染 HTML。

### 1.1 强制技术栈

| 规则 | 说明 |
|------|------|
| **Table-based 布局** | 无 `<div>` 布局、无 Flexbox、无 CSS Grid |
| **Inline CSS** | 所有样式内联写在 `style` 属性。`<style>` 块仅作 Gmail fallback |
| **宽度 ≤ 600px** | 容器固定 600px 居中。移动端 `max-width: 100%` 退化 |
| **无 JavaScript** | 零 JS，不支持 onclick、动画、交互 |
| **无外部资源依赖** | CSS Reset、Tailwind、Google Fonts 均不可用 |

### 1.2 字体与背景

- 标题：`Georgia, 'Times New Roman', serif`
- 正文：`Arial, 'Helvetica Neue', Helvetica, sans-serif`
- 背景色用 `bgcolor` HTML 属性，不是 CSS `background-color`
- 不要用背景图片（Outlook 不支持）
- 图片必须写 `width`、`height`、`alt`、`border="0"`

### 1.3 CTA 按钮方案

邮件客户端对 `border-radius` 支持不一致。推荐全宽 Table Cell 方案：

```html
<table width="100%" cellpadding="0" cellspacing="0" border="0">
  <tr>
    <td align="center" style="background-color:#1a1a2e; padding:14px 28px; border-radius:6px;">
      <a href="{{url}}" style="color:#ffffff; text-decoration:none; font-family:Arial,sans-serif; font-size:16px; font-weight:600;">
        立即开始
      </a>
    </td>
  </tr>
</table>
```

Outlook 圆角需求时用 VML `<v:roundrect>`（见反模式速查）。

## 2. 冷邮件设计规则

冷邮件 = 第一次接触。目标不是展示设计，而是被打开、被阅读、被点击。

### 2.1 旋钮预设

| 旋钮 | 值 | 含义 |
|------|-----|------|
| DESIGN_VARIANCE | 1-2 | 极低实验性，可预测，专业 |
| MOTION_INTENSITY | 1 | 纯静态 |
| VISUAL_DENSITY | 2-3 | 克制信息密度，留白呼吸 |

### 2.2 结构规则

| 元素 | 约束 |
|------|------|
| 主题行 | ≤ 60 字符（手机端显示约 30-40 字符）。不要 Re: 伪装、不要全大写、不要 emoji |
| 预览文本 | ≤ 100 字符。补充主题行，不要重复 |
| 正文 | ≤ 150 词。冷邮件不是长文 |
| CTA | 一个 CTA，一个意图。不要多个按钮竞争 |
| 发件人 | 真人姓名 + 职位。不要 `noreply@` 或 `team@` |
| 个性化 | 用 `{{first_name}}`、`{{company_name}}`。至少 1 个个性化字段 |

### 2.3 视觉层级

冷邮件的视觉层级与网页相反。网页从上到下扫视；邮件在预览窗格中**只有一屏**。

1. **发件人姓名 + 主题行** = 80% 的打开率决定因素
2. **第一句话** = 80% 的阅读决定因素
3. **CTA** = 80% 的点击决定因素

- 第一句话必须在预览窗格可见，不需要滚动
- CTA 必须在第一屏内
- 不要在 CTA 之前放长段背景介绍

## 3. 营销邮件规则

### 3.1 旋钮预设

| 旋钮 | 值 | 含义 |
|------|-----|------|
| DESIGN_VARIANCE | 2-3 | 低实验性，品牌叙事为主 |
| MOTION_INTENSITY | 1 | 纯静态 |
| VISUAL_DENSITY | 3-4 | 标准密度 |

### 3.2 四层结构

```
┌──────────────────────┐
│   1. 主图 / Hero     │  600×300 图片或品牌色块 + 大标题
├──────────────────────┤
│   2. 标题 + 副标题    │  核心信息，≤ 2 行标题 + ≤ 3 行副标题
├──────────────────────┤
│   3. 正文 / 内容块     │  产品信息、活动详情
├──────────────────────┤
│   4. CTA             │  一个主按钮 + 一个次链接（可选）
└──────────────────────┘
```

- 主图宽度 = 容器宽度（600px），高度 200-300px
- 标题 22-28px 加粗，正文 14-16px 行高 1.5
- CTA 居中，颜色 = 品牌强调色

### 3.3 AI 痕迹规避

| 痕迹 | 修复 |
|------|------|
| 深蓝 Header + 灰底 Footer（AI 默认） | 用品牌色，不是默认深蓝 |
| Em-dash `—` 在标题或分隔线中 | 用水平线 `<hr>` 或留白分隔 |
| 一封邮件塞 5+ 内容块 | 一封一个核心主题，多主题拆多封 |
| 三列等宽产品卡照搬网页 | 邮件 ≤ 2 列，优先单列 |

### 3.4 A/B 测试要点

每次只测一个变量：
- 主题行：利益驱动 vs 好奇驱动
- 发件人：CEO 姓名 vs 产品经理姓名
- 发送时间：周二 10:00 vs 周四 15:00
- CTA 文案：具体动词 A vs 具体动词 B
- CTA 位置：标题下方 vs 正文末尾

## 4. 事务邮件规则

事务邮件 = 订单确认、报价单、密码重置、通知。

### 4.1 旋钮预设

| 旋钮 | 值 |
|------|-----|
| DESIGN_VARIANCE | 1-2 |
| MOTION_INTENSITY | 1 |
| VISUAL_DENSITY | 4-5（高信息密度） |

### 4.2 核心原则

| 原则 | 说明 |
|------|------|
| **简洁优先** | 去掉装饰性元素，事务邮件不是营销场 |
| **信息密度** | 用户打开是为获取信息，不是欣赏设计 |
| **行动导向** | 每封有明确下一步（确认订单、修改密码、查看详情） |
| **可扫描** | 关键数据（订单号、金额、日期）加粗或色块突出 |

## 5. Onboarding 序列规则

### 5.1 序列规划

| 规则 | 说明 |
|------|------|
| **封数** | 3-5 封，超过 5 封 = 打扰 |
| **节奏** | 注册即发第 1 封，之后每 1-3 天一封 |
| **递进叙事** | 每封在上一封基础上深入 |
| **一个核心动作** | 每封只引导用户做一件事 |

### 5.2 标准序列结构

| # | 时机 | 主题 | 核心动作 |
|---|------|------|---------|
| 1 | 注册后立即 | 欢迎 | 完成个人资料 / 首次登录 |
| 2 | 第 1-2 天 | 核心功能引导 | 使用第一个关键功能 |
| 3 | 第 2-4 天 | 进阶技巧 | 使用进阶功能 |
| 4 | 第 3-7 天（可选） | 社交证明 | 邀请同事 / 查看案例 |
| 5 | 第 7-14 天（可选） | 转化推进 | 升级 / 续费 / 联系销售 |

### 5.3 序列一致性

- **品牌色**全序列一致，不要每封换配色
- **发件人**同一人或同品牌下不同真人
- **Logo** 每封头部放品牌 Logo，相同尺寸和位置
- **Footer** 退链 + 公司信息，全序列统一

## 6. 邮件品牌一致性

从官网设计系统提取邮件 Token：

| 网站 Token | 邮件适配 |
|------------|---------|
| 品牌主色 | CTA `background-color` + 品牌色块 |
| 品牌强调色 | 链接颜色、小图标色 |
| 品牌中性色 | 正文颜色、背景色 |
| 字体栈 | Web-safe 等价字体（Arial 代替品牌无衬线，Georgia 代替品牌衬线） |
| Logo | 同一文件、相同比例 |

## 7. 邮件测试检查清单

### 7.1 渲染测试

| 客户端 | 优先级 | 重点 |
|--------|--------|------|
| Gmail（网页 / iOS / Android） | P0 | 图片显示、链接、主题行截断 |
| Outlook（Windows 桌面版） | P0 | VML 按钮、table 布局、bgcolor |
| Outlook（macOS） | P1 | 布局、字体回退 |
| Apple Mail（macOS / iOS） | P1 | 布局、字体渲染 |
| QQ 邮箱 / 网易邮箱 | P2（国内项目 P0） | 中文字体渲染 |

### 7.2 内容检查

- [ ] 主题行 ≤ 60 字符
- [ ] 预览文本已设置
- [ ] 所有链接可点击且指向正确
- [ ] CTA 触摸区域 ≥ 44×44px
- [ ] 所有 `<img>` 有 `alt`、`width`、`height`
- [ ] 无 emoji（渲染不一致）
- [ ] 退链链接存在且功能正常
- [ ] 个性化变量不会导致空值破版

### 7.3 其他必做项

- **纯文本备选**：HTML 邮件必须提供纯文本版本
- **深色模式声明**：`<meta name="color-scheme" content="light">` 避免客户端自动反色
- **预渲染截屏**：用 Litmus / Email on Acid 生成各客户端截图，逐个检查

## 8. 反模式速查

| 反模式 | 正确做法 |
|--------|---------|
| `<div>` 布局 | 所有布局用 `<table>` |
| CSS `float` | 用 table `align` 和 `valign` |
| `<style>` 写所有样式 | 关键样式 inline，`<style>` 做 fallback |
| 600px 容器内嵌 1200px 图片 | 图片设 `width="100%"` |
| 多个 CTA 竞争 | 一封邮件一个主 CTA |
| 全大写主题行 | 首字母大写，自然语感 |
| 背景图片 + 上叠文字 | 实色背景 + `<img>` |
| Re: 伪装回复 | 诚实主题行 |
| 邮件中放视频/GIF | 静态图 + 播放按钮样式链接到落地页 |
| 过度设计事务邮件 | 信息优先，装饰克制 |
| 深蓝 Header + 灰底 Footer | 用品牌色 |
