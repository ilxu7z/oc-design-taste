# 生图规则（Image Generation Domain）

> 本文件覆盖：Hero 图、场景应用图、痛点概念图、品牌套件（Brandkit）的完整生成纪律。
> 融合自 design-taste v4（Section 2.C / 4.5 / 10）、原版 brandkit（798 行完整版）、imagegen-frontend-web 排版规则与构图多样性强制。

---

## 1. 生图工作流纪律

### 1.1 铁律：Agent 禁止自己写提示词

```
GLM-5.1 → 生图模型 → Agent 审查
```

**流程：**
1. **GLM-5.1 写提示词** — Agent 将生图需求（场景描述、构图要求、品牌色 hex、光源方向、禁止项）交给 GLM-5.1，由 GLM-5.1 生成完整提示词
2. **生图模型执行** — 将 GLM-5.1 的提示词传入生图模型（Midjourney / DALL-E / Stable Diffusion / Flux）
3. **Agent 审查** — Agent 用 AI 生图痕迹检测表（Section 2）逐条审查输出，不合格则打回重生成

**禁止行为：**
- ❌ Agent 自己写提示词直接喂给生图模型
- ❌ 跳过 GLM-5.1 直接生图
- ❌ 审查时跳过痕迹检测表

**突破条件：** 无。这是 MEMORY.md 的铁律。

### 1.2 生图前必读信号

每次生图前，先输出一行 Design Read（参照 design-taste Section 0.B）：

```
「生图类型」×「受众」×「调性方向」
→ 美学方向：[选定方向]
→ 技术参数：[尺寸/比例/光源/色温]
```

示例：
```
产品 Hero 图 × 海外B端买家 × 洁白专业
→ 美学方向：商业摄影，studio lighting，浅景深，实拍质感
→ 技术参数：16:9，300dpi，主光源左上45°，色温5500K

场景应用图 × 东南亚终端消费者 × 自然温暖
→ 美学方向：生活方式，自然光线，真实环境，有温度
→ 技术参数：4:3，自然光+补光，色温4000K，浅景深
```

### 1.3 生图资产优先级链

| 优先级 | 来源 | 适用场景 |
|--------|------|----------|
| **1. 品牌实际图片** | 用户提供的 URL/文件 | 最佳——永远优先用真实素材 |
| **2. 生图工具** | GLM-5.1 prompt → 生图模型 | Hero、场景图、产品特写 |
| **3. Picsum-seed** | `picsum.photos/seed/{描述}/{w}/{h}` | 占位图，seed 必须描述内容 |
| **4. 明确标注占位** | `<!-- TODO: hero photo 1600x1200 -->` | 前三者都不可用时 |

**❌ 永远不做：** 用 `<div>` 矩形模拟截图、手绘 SVG 图标当装饰、纯文字页面称"极简"。

## 2. AI 生图痕迹检测表（6 条强制审查）

> 每次生图输出后，Agent 必须逐条审查。任何一条不合格 → 打回重生成。

| # | 痕迹 | 说明 | 处理方式 | 提示词修复示例 |
|---|------|------|---------|--------------|
| 1 | ❌ AI 完美感 | 无尘、对称、过度饱和、塑料感、完美无瑕 | prompt 加 "workshop feel, natural arrangement, realistic imperfections, slightly asymmetrical" | "workshop feel, natural arrangement, slight dust, realistic lighting, not perfectly symmetrical" |
| 2 | ❌ 面孔不匹配 | 东南亚内容用白/黑人面孔，或面孔风格与场景矛盾 | prompt 加 "Southeast Asian faces" 或具体族群描述 | "Southeast Asian male, 30s, casual work attire, natural expression, not model-like" |
| 3 | ❌ 空洞背景 | 无环境上下文，产品悬浮在虚无中 | ZONAL ANALYSIS 填满 Zone A/B/C/D（前/中/背景/装饰层） | "kitchen counter with utensils in background, wooden surface, natural afternoon light, herbs in frame" |
| 4 | ❌ 幻觉文字 | 画面内出现不存在的中英文、乱码字符、假招牌 | prompt 注明 "no visible text or labels, no signage, no written characters" | "no visible text, no labels, no signage, no written characters anywhere in the image" |
| 5 | ❌ 光源无方向 | 产品图无主光源信息，均匀照明导致无立体感 | 标注光源方向/色温/景深 | "directional light from upper left at 45 degrees, warm 4000K, shallow depth of field, distinct shadow on right side" |
| 6 | ❌ 品牌色缺失 | 生图不带品牌色融入，画面色彩与品牌无关 | prompt 标注品牌色 hex，要求融入环境 | "brand color #2D5A27 (forest green) as accent in background elements, #F5F0EB as dominant surface tone" |

### 2.1 检测执行流程

```
生图输出 → 逐条过 6 条检测表 → 全部通过 → 交付
                                  → 任意不合格 → 分析原因 → 补充 GLM-5.1 修复提示词 → 重生图 → 重新审查
```

**审查不通过时，不要自己改提示词。** 把修复要求给 GLM-5.1，由 GLM-5.1 重写提示词。

## 3. Hero 图构图规则

### 3.1 尺寸与比例

| 场景 | 推荐比例 | 分辨率 | 说明 |
|------|---------|--------|------|
| 官网 Hero（桌面） | 16:9 或 21:9 | 1920×1080 或 2560×1097 | 宽屏电影感 |
| 官网 Hero（响应式） | 16:9 + 4:3 备选 | 1920×1080 + 1200×900 | 桌面+平板两版 |
| 产品 Hero 图 | 4:3 或 3:2 | 2400×1800 或 2400×1600 | 商业摄影比例 |
| 画册封面 Hero | 4:3 或 A4 比例 | 300dpi 对应 | 印刷级 |
| 品牌 Hero 海报 | 2:3 或 9:16 | 竖版 | 社交/海报用 |

### 3.2 品牌元素融入规则

| 规则 | 说明 | 突破条件 |
|------|------|---------|
| 品牌色必现 | 至少 1 个品牌色在画面中作为 accent（道具/背景/光线/材质） | 品牌色与场景完全不兼容时用中性色过渡 |
| Logo 克制 | Hero 中 logo 不超过画面 5% 面积，不遮挡主体 | 品牌封面/Logo 展示页 |
| 品牌质感 | 材质/光影风格匹配品牌调性（工业=金属质感，食品=自然质感） | 无 |
| 留白预留 | 文字区域预留干净的负空间（左上/左下/中心低区） | Editorial 全幅画面 |
| 景深控制 | 主体清晰，背景柔和虚化，模拟真实摄影 | 产品平铺图 |

### 3.3 Hero 文字区域预留

Hero 图必须为标题、副标题、CTA 预留干净的负空间区域：

| 文字位置 | 适用场景 | 预留要求 |
|---------|---------|---------|
| 左下区 | 大多数 Hero | 画面左下 40% 区域保持低对比度/纯色/虚化 |
| 中心低区 | 电影感 Hero | 画面下半部 30% 区域干净 |
| 左上区 | 右图左文布局 | 画面左上 35% 区域干净 |
| 全幅叠加 | 极简 Hero | 整体低对比度 overlay，文字可读 |

### 3.4 Hero 构图多样性强制

**不要默认左文右图。** 每次选择 Hero 构图时，从以下方案中选一个（用用户 prompt 字符数做 seed 模拟随机选择）：

1. **左文右图（经典）** — 文字在左 40%，图片在右 60%
2. **右文左图（反转经典）** — 文字在右 40%，图片在左 60%
3. **中心低区叠加** — 全幅背景图，文字在画面下半部 40% 区域
4. **左上引导** — 文字左上，图片占据剩余画面
5. **左下叠加** — 全幅背景图，文字在左下角
6. **右下叠加** — 全幅背景图，文字在右下角
7. **居中叠加** — 全幅背景图，文字居中，极简
8. **分屏** — 上下分屏或左右分屏，文字和图片各占一半
9. **全幅文字 + 图片 inset** — 大字排版为主，图片作为 inset 嵌入
10. **图片作为画布** — 图片充满全屏，文字在干净 safe area 内叠加

**铁律：** 连续两次输出不能选同一个构图方案。

## 4. 场景应用图规则

### 4.1 产品使用场景图

| 要素 | 要求 | 避免 |
|------|------|------|
| 真实环境 | 产品在真实使用环境中（厨房/车间/办公室/户外） | 纯色背景/摄影棚白底 |
| 真人操作 | 有人手/人在使用产品，自然姿态 | 产品悬浮无人 |
| 环境上下文 | 环境细节丰富但不杂乱 | 空洞背景 |
| 自然光线 | 模拟自然光或场景合理光源 | 均匀无方向照明 |
| 产品突出 | 产品在画面中视觉焦点明确 | 产品被环境淹没 |
| 品牌色融入 | 品牌色在道具/环境/服装中自然出现 | 品牌色生硬插入 |

### 4.2 真人面孔规则

| 规则 | 说明 |
|------|------|
| 面孔匹配 | 内容面向东南亚 → 东南亚面孔；面向欧美 → 欧美面孔 |
| 自然表情 | 不笑或微笑，不要夸张大笑/惊讶 |
| 年龄匹配 | 场景合理年龄（B端采购=30-50岁，消费品=20-40岁） |
| 服装匹配 | 服装符合场景（工厂=工装，办公室=商务休闲） |
| 不要模特感 | 避免完美妆容、摆拍感、模特卡姿势 |

### 4.3 环境要求

| 场景类型 | 环境要求 |
|---------|---------|
| 厨房/餐饮 | 有食材/餐具/操作痕迹，不要空台面 |
| 工厂/车间 | 有工具/材料/操作细节，不要空厂房 |
| 办公室 | 有文档/设备/办公用品，不要空桌子 |
| 户外 | 有自然元素/天气暗示，不要纯蓝天绿地 |
| 零售/店铺 | 有商品/价签/顾客，不要空货架 |

### 4.4 场景应用图构图规则

- **三分法**：主体在画面 1/3 或 2/3 位置
- **引导线**：利用环境线条（桌面边缘/道路/管道）引导视线到产品
- **前景层**：增加前景虚化元素增加景深感
- **色彩和谐**：环境色与品牌色协调，不冲突
- **不要：** 产品居中正对镜头、对称构图、纯色背景

## 5. 痛点概念图规则

### 5.1 连锁餐饮痛点等抽象概念可视化

痛点概念图用于可视化抽象问题（效率低、成本高、管理混乱、食品安全等）。

| 痛点类型 | 视觉策略 | 示例 |
|---------|---------|------|
| 效率低 | 对比：混乱 vs 有序、慢 vs 快 | 杂乱厨房 vs 整洁流水线 |
| 成本高 | 视觉堆叠：重复元素表示浪费 | 堆积的账单/食材浪费 |
| 管理混乱 | 视觉噪音：多线交叉/信息过载 | 缠绕的线/重叠的表格 |
| 食品安全 | 对比：脏 vs 干净、危险 vs 安全 | 模糊脏污 vs 清晰洁净 |
| 数据不透明 | 迷雾/遮挡/模糊 | 模糊的数字/被遮挡的屏幕 |

### 5.2 痛点图视觉语言

| 要素 | 要求 |
|------|------|
| 强对比 | 用对比色（红/橙 vs 绿/蓝）表达问题 vs 解决方案 |
| 夸张表现 | 适度夸张使问题视觉化，但不失真 |
| 视觉叙事 | 一张图讲一个故事：问题 → 后果 → 暗示解决方案 |
| 品牌色融入 | 解决方案侧融入品牌色，问题侧用中性/冷色 |
| 不要 | 过于抽象无法理解、过于血腥/负面、纯文字图表 |

### 5.3 痛点图构图

- **左右对比**：左侧问题，右侧解决方案（或反之）
- **上下对比**：上方问题，下方解决方案
- **中心聚焦**：问题在中心，解决方案元素从边缘渗透
- **渐变过渡**：从左到右从问题色过渡到品牌色
- **不要：** 问题侧太恐怖/恶心、解决方案侧太完美不真实

### 5.4 痛点概念图旋钮值

参照 design-taste Section 1 场景预设：

| 参数 | 值 | 说明 |
|------|-----|------|
| DESIGN_VARIANCE | 7-8 | 允许强对比和夸张 |
| MOTION_INTENSITY | 1 | 静态图 |
| VISUAL_DENSITY | 5-6 | 信息密度适中偏强 |

## 6. 完整品牌套件规则（Brandkit）

> 从原版 brandkit（798 行）完整融合。用于品牌标识系统、Logo 概念板、品牌手册封面等视觉交付。

### 6.1 核心原则

品牌套件不是装饰。它是品牌为何存在的视觉论证。每张生成板必须回答：
1. 这个品牌代表什么？
2. 核心隐喻是什么？
3. Logo 如何表达这个隐喻？
4. 系统如何在 UI、印刷、图片、细节上扩展？
5. 为什么整体感觉是可拥有的？

### 6.2 品牌策略推断

生成前推断品牌策略。思考：品类、受众、产品功能、情感承诺、文化定位、信任层级、视觉世界、符号隐喻、品牌应避免什么。

| 类别 | 核心想法 | 可能的符号逻辑 |
|------|---------|--------------|
| 开发者工具 | 构建、速度、精确、控制 | 光标、框架、螺栓、脚手架、网格 |
| AI 助手 | 委托、智能、清晰 | 火花、轨道、信号、路径、节点 |
| 安全 | 保护、警戒、边界 | 盾牌、眼睛、印章、受保护核心 |
| 游戏/博彩 | 机会、回报、紧张、速度 | 骰子、宝石、卡片、信号、奖杯 |
| 语音 AI | 声音、节奏、命令、流动 | 波形、麦克风、球体、语音路径 |
| 合规 | 信任、秩序、规则、保护 | 印章、狗、徽章、文档、盾牌 |
| 机器人/无人机 | 飞行、控制、视觉、任务 | 翅膀、猫头鹰、准星、路径、区域 |
| 奢侈品/编辑 | 品味、材料、仪式、克制 | 字母组合、印章、纸张、压印、标记 |
| 生产力 | 专注、动量、清晰 | 路径、勾选、方块、日历、光 |

**不要随机选符号。** 符号必须基于品牌意义。

### 6.3 Logo 概念方法（选 1-2 种）

#### 方法 1：Monogram + Meaning（字母组合 + 隐喻）

品牌首字母与隐喻结合。示例：
- `K` + 风筝/框架/方向
- `N` + 路径/折叠系统
- `S` + 声波/语音流
- `A` + 上升/建筑/动量

不要做无聊的字母图标。用负空间、切割、折叠或几何。

#### 方法 2：Product Action（产品动作）

将产品的主要动作转为符号。示例：
- build → 框架、脚手架、方块、光标
- protect → 盾牌、边界、警戒标记
- convert → 开关、箭头、变形形状
- speak → 波形、麦克风、脉冲
- hunt threats → 眼睛、猛禽、雷达、轨迹
- automate → 循环、交接、路径

使其抽象且高级，不要过于具象。

#### 方法 3：Metaphor Fusion（隐喻融合）

将两个有意义的想法融合为一个精简标记。示例：
- 猫头鹰 + 无人机视觉
- 盾牌 + 山
- 月亮 + 波形
- 狗 + 合规印章
- 骰子 + 手机游戏经济
- 光标 + 闪电速度
- 风筝 + 产品框架

融合应微妙且可读。

#### 方法 4：Negative Space（负空间）

用空白创造智能。示例：
- 隐藏的箭头
- 受保护的中心
- 切割首字母
- 内部路径
- 折叠角落
- 交叉形状形成的眼睛

负空间应清晰锐利。

#### 方法 5：Construction Geometry（构造几何）

从清晰系统创建标记。使用：
- 圆
- 对角线切割
- 网格
- 框架
- 模块化方块
- 分层卡片
- 轨道路径
- 准星
- 测量线条

一个面板可展示构造逻辑。

### 6.4 默认 3×3 面板系统

如未指定布局，使用以下 9 面板系统：

#### 面板 1：Logo Cover（Logo 封面）
大 Logo + 字标。极简标题。强负空间。

#### 面板 2：Logo Construction（Logo 构造）
符号分解、网格、几何或负空间逻辑。展示标记为何存在。

#### 面板 3：Digital Application（数字应用）
浏览器 chrome、app header、终端、仪表盘片段或 app 图标。

#### 面板 4：Brand Essence（品牌本质）
一句短标语。大号可读排版。稀疏构图。

#### 面板 5：Color System（色彩系统）
色板、渐变条、色盘、材质芯片或调色板卡片。

#### 面板 6：Typography（排版）
大字型样本、字母行或主/副字体配对。

#### 面板 7：Physical Application（实体应用）
卡片、文件夹、徽章、海报、标签、印章、包装或物体 mockup。

#### 面板 8：Image Direction（图片方向）
电影景观、产品裁剪、半色调海报、编辑场景、材质纹理。

#### 面板 9：System Detail（系统细节）
UI 芯片、输入栏、命令行、图标行、徽章系统、组件条、图案细节。

### 6.5 2×3 参考风格布局

对于参考风格的品牌板：

1. **Logo / 字标** — 居中或偏移，极度极简
2. **浏览器 / 产品表面** — 浏览器栏、app 框架、prompt 输入或 URL 字段
3. **命令 / 功能面板** — 终端、prompt 栏、输入状态、安装命令、仪表盘片段
4. **氛围 / 广告图** — 半色调景观、电影感图片、产品世界视觉或艺术指导照片
5. **符号 / 构造 / 徽章** — 准星中的 logo 标记、印章、几何框架、图标构造
6. **标语 / 系统承诺** — 一行短文字，大号排版，安静背景

### 6.6 8 种视觉模式完整描述

#### 模式 1：Dark Developer（暗色开发者）
**适用：** 开发者工具、编码 Agent、基础设施、自动化、AI 构建者
**视觉线索：**
- 近黑面板
- 等宽强调
- 命令行
- 终端窗口
- prompt 栏
- 微妙网格
- 青色/蓝色/珊瑚/石灰强调
- 如合适可用像素或 CRT 纹理

标语应简短且具体。

- 暗色产品特写
- 戏剧性物体裁剪
- 纹理纸背景

- 抽象但受控的视觉系统

**避免：**
- 通用库存人物
- 随机办公室照片
- 陈词滥调的机器人图像
- 过于繁忙的场景
- 无关图像

图片应匹配调色板和隐喻。

### 6.18 Logo Generation Standard（Logo 生成标准）

Logo 必须专业。

应：

- 令人难忘

Logo 应感觉来自研究和简化。

## 7. 构图多样性强制

> 从 imagegen-frontend-web 移植。防止 AI 默认输出"左文右图"的惯性。

### 7.1 Hero 构图多样性

每次生 Hero 图前，从以下 10 种方案中选一个。**连续两次不能选同一个。**

1. **左文右图（经典）** — 文字左 40%，图片右 60%
2. **右文左图（反转经典）** — 文字右 40%，图片左 60%
3. **中心低区叠加** — 全幅背景图，文字在画面下半部 40%
4. **左上引导** — 文字左上，图片占据剩余画面
5. **左下叠加** — 全幅背景图，文字在左下角
6. **右下叠加** — 全幅背景图，文字在右下角
7. **居中叠加** — 全幅背景图，文字居中极简
8. **分屏** — 上下或左右分屏，文字和图片各半
9. **全幅文字 + 图片 inset** — 大字排版为主，图片嵌入
10. **图片作为画布** — 图片全屏，文字在干净 safe area 内叠加

### 7.2 Section 构图多样性

对于多 section 生图（如品牌套件 9 面板），确保：
- 至少 3 种不同构图锚点出现在不同面板中
- 同一构图锚点不连续出现超过 2 次
- 至少 1 个面板使用全幅背景模式（非纯色 inline asset）

### 7.3 背景模式多样性

每个面板/图片选择不同的背景模式，确保全页不单调：

| 模式 | 适用场景 |
|------|---------|
| 纯色表面 + inline 资产 | 功能面板、系统细节 |
| 微妙纹理/纸张/网格 | 排版面板、品牌本质 |
| 全幅图片背景 + 色调覆盖 | Logo 封面、氛围面板 |
| 编辑侧图（50/50 或 60/40） | 数字应用、图片方向 |
| 图片作为整个视觉 + 文字叠加 | 氛围面板 |
| 纯色色块 + 小产品/细节裁剪 | 实体应用 |
| 电影色调渐变（调色板匹配） | 品牌封面 |
| 双色调处理图片 | 编辑风格面板 |
| 柔和径向晕影 + 产品裁剪 | 奢侈品风格 |
| 微噪点渐变叠加纯色 | 高级触感深度 |

### 7.4 多样性检查清单

每次交付前检查：
- [ ] Hero 构图不是默认左文右图？
- [ ] 连续两次输出构图不同？
- [ ] 多面板项目中至少 3 种不同构图锚点？
- [ ] 同一构图锚点不连续出现超过 2 次？
- [ ] 背景模式有变化（不全是纯色表面）？

## 8. 面孔 & 地名真实性

- 不用 John Doe / Sarah Chan / Jack Su → 用真实感名称
- 面孔匹配内容面向地区（东南亚/欧美/中东等）
