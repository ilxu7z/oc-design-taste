## 设计系统选型铁律

### 铁律 0：区分"官方系统"和"美学方向"

| 类型 | 含义 | 例子 | 处理方式 |
|------|------|------|---------|
| **官方设计系统** | 有 npm 包、文档、版本号 | Fluent UI、Carbon、Material Web 3、Polaris | 安装官方包，不手写 |
| **美学方向** | 视觉风格描述，无官方实现 | Glassmorphism、Bento Grid、Brutalism、Neumorphism、Apple Liquid Glass | 用 Tailwind + 自建组件实现近似效果 |

⚠️ **Apple Liquid Glass 明确标注：** Apple 没有发布官方的 `liquid-glass.css` 或任何 Web 实现。所有 Web 上的 "Liquid Glass" 效果都是社区近似实现（backdrop-blur + saturate + 多层半透明叠加）。不要声称"官方 Liquid Glass"。

### 铁律 1：一个项目只用一种设计系统

不混搭 Material + shadcn，不混 Fluent + Carbon 在同一组件树。

### 铁律 2：用官方包，不手写

如果 Brief 对应某个设计系统，安装并使用**官方**包。不要手写其 CSS。不要导入系统 token 然后覆盖 90%。

### 铁律 3：依赖验证

引入任何第三方库前，先检查 `package.json`。缺少则先输出安装命令。**永远不要假设库已存在。**

### 选型速查

| Brief 信号 | 选什么 | 为什么 |
|-----------|--------|--------|
| 自己的 Tailwind 项目 | Tailwind v4 + 自建组件 | 已有体系，不引入第三方 DS |
| "干净/极简/Linear 风" | Tailwind + Geist / Outfit 字体 | 轻量，完全自主控制 |
| "信任优先/严肃 B端" | Tailwind + 紧凑间距 + 少动效 | 不需要重型设计系统 |
| "Material 风格" | Material Web 3 | 仅在用户明确要求时 |
| "公共部门/政府" | 参考 GOV.UK 布局原则 | 仅需原则，不安装库 |
| "Microsoft/企业SaaS" | Fluent UI React | 官方 Fluent，无障碍完成 |
| "IBM 风格 B端" | Carbon React | 官方 Carbon，成熟数据密度模式 |
| "Atlassian 风格" | Atlaskit | 官方 Atlaskit |
| "GitHub 风格" | Primer CSS | 官方 Primer |
| "现代 SaaS 自建组件" | shadcn/ui | 代码归你，易定制；从不发默认状态 |
| "Shopify 应用" | Polaris | 官方 Polaris，Shopify 生态必需 |
| "框架无关组件" | Fluent Web Components | 跨框架企业应用 |
| "GitHub 营销页" | Primer Brand | 官方品牌组件 |
| "无障碍 React 原语" | Radix UI | 无样式，完全控制 |
