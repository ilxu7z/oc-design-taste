# 色彩科学知识库

> 融合 oc v5 Section 4.2 和原版 taste-skill Section 4.2 的色彩规则。
> 核心原则：AI 输出的默认色彩 = 模板感。主动覆盖这些偏差。

---

## 1. 基础纪律

| 规则 | 说明 | 突破条件 |
|------|------|---------|
| 单一强调色 | 全页只用 1 个 accent color | 品牌本身有多色体系且用户确认 |
| 饱和度 < 80% | B端 < 60%，消费者 < 80% | 品牌色本身就是高饱和（如可口可乐红） |
| 单一灰色系 | Zinc/Slate/Stone 择一，全页一致 | 无 |
| 不用纯黑 #000 + 纯白 #FFF | 用 #111 + #FAFAFA 或品牌色系 | 纯黑用于印刷（CMYK 100%K） |
| Color Consistency Lock | 选定 accent 后，全页所有 section 一致 | 无——这是铁律 |
| 阴影调色 | 阴影色调匹配背景色，不用纯黑 drop shadow | 无 |
| 光源方向一致 | 全页阴影暗示同一光源方向 | 无 |

## 2. ⚠️ Premium Consumer 色板禁令

### 2.1 为什么禁

这是 AI 第二大色板惯性。高端消费品（厨具/保健品/奢侈品/手工艺/DTC家居），AI 默认输出同一套暖米咖喱色板。每个 AI 生成的消费品品牌看起来都一样。

### 2.2 禁止的 hex 色值（完整列表）

**背景类（暖米/奶油/粉笔/骨色）：**
```
#f5f1ea  #f7f5f1  #fbf8f1  #efeae0
#ece6db  #faf7f1  #e8dfcb  #f5f0eb
```

**强调色（黄铜/陶土/牛血红/赭色）：**
```
#b08947  #b6553a  #9a2436  #9c6e2a
#bc7c3a  #7d5621  #8b4513  #a0522d
```

**文字色（浓缩咖啡近黑）：**
```
#1a1714  #1a1814  #1b1814  #2c1810
```

### 2.3 禁令规则

除非品牌本身是暖色调/复古工艺定位，否则不要默认 reach for 这个色板。**轮换规则：** 如果上一个高端消费品项目用了暖米+黄铜家族，这个项目必须用不同的色板。

### 2.4 替代方案（6-7 种，轮换使用）

| # | 色板名称 | 描述 | 风格参考 |
|---|---------|------|---------|
| 1 | **冷奢（Cold Luxury）** | 银灰 + 铬色 + 烟熏，冷调高级 | Tesla、Apple Watch Hermes（无皮革版） |
| 2 | **森林（Forest）** | 深绿 + 骨白 + 琥珀点缀，自然 premium | Filson、Patagonia premium |
| 3 | **蓝铜（Cobalt + Cream）** | 钴蓝 + 奶油单一中性色，无黄铜 | 现代科技消费品 |
| 4 | **陶土石板（Terracotta + Slate）** | 暖锈红 + 冷灰，无黄铜 | 手工品牌、家居 |
| 5 | **黑褐（Black and Tan）** | 纯近黑 + 暖褐，高对比，无米色 | 极简高端品牌 |
| 6 | **纯单色 + 单一饱和点缀** | 近白 + 近黑 + 一个亮色（电蓝/翠绿/热粉） | 现代时尚品牌 |
| 7 | **橄榄砖红（Olive + Brick + Paper）** | 哑光橄榄绿 + 砖红点缀 + 纸白背景 | 自然有机品牌 |

### 2.5 轮换规则

- 不连续两个项目用同一色板
- 记录上次使用的色板，下次强制轮换
- 突破条件：品牌 Brief 明确指定颜色

## 3. ⚠️ 紫色/蓝色 AI 渐变禁令（The Lila Ban）

### 3.1 禁令描述

"AI 紫/蓝发光"审美默认禁止。不要自动用紫色按钮发光、随机霓虹渐变。用中性基色（Zinc/Slate/Stone）+ 高对比单一强调色（翠绿、电蓝、深玫红、焦橙等）。

### 3.2 为什么禁

这是 LLM 最爱的"科技感"伪装。几乎所有 AI 生成的科技/SaaS 页面都默认使用紫蓝渐变 Hero，形成强烈的模板指纹。

### 3.3 突破条件

品牌或 Brief 明确要求紫色/紫罗兰色。但执行要刻意：一致调色板、协调中性色、克制渐变。

### 3.4 AI 紫蓝渐变指纹识别

以下模式是 AI 紫蓝痕迹的标志：

- Hero 背景：深紫到亮蓝的径向渐变（`#4F46E5 → #7C3AED` 或类似）
- 按钮：紫色发光 + 白色文字
- 装饰：霓虹紫 glow 线条、紫色模糊圆点
- 文字：紫色渐变标题（`bg-gradient-to-r from-purple-600 to-blue-500 bg-clip-text text-transparent`）

**检测到以上任意模式 → 必须替换为中性基色 + 单一强调色。**

## 4. Em-Dash 禁令

### 4.1 全面禁令

**Em-dash（`—`）和 En-dash（`–`）在所有可见文本中完全禁止。**

这是 LLM #1 文案痕迹。AI 用 em-dash 当万能分隔符、修辞手段、设计元素。

### 4.2 替代方案

| 禁止位置 | 替代方案 |
|---------|---------|
| 标题 | 用句号或逗号 |
| 副标题/标签/按钮/图片说明/导航 | 用换行、竖线或空格 |
| 正文 | 拆成两句话，或用逗号/括号/冒号 |
| 引用归属 | 用普通连字符 ` - ` 或换行 + 小字 |

### 4.3 唯一允许的 dash

普通连字符 `-`（复合词、数字范围）。零 em-dash，零例外。

## 5. 色彩一致性规则

### 5.1 不混冷暖灰

选定一个灰色系（Zinc/Slate/Stone）后全页一致：

| 灰色系 | 色调 | 适用场景 |
|--------|------|---------|
| **Zinc** | 中性灰，偏冷 | 通用默认，B端，SaaS |
| **Slate** | 冷灰，带蓝底 | 科技品牌，开发者工具 |
| **Stone** | 暖灰，带褐底 | 消费品，编辑排版 |
| **Neutral** | 纯中性灰 | 公共部门，无障碍优先 |

**铁律：** 一个项目只用一种灰色系。不混 Zinc 标题 + Stone 背景。

### 5.2 不超过 1 个 accent

- 全页只用 1 个强调色
- 强调色用于：CTA 按钮、链接、图标、关键数据高亮
- 不用于：背景大面积填充、装饰渐变、随机元素
- 突破条件：品牌本身有多色体系且用户确认

### 5.3 Color Consistency Lock

选定 accent 后，全页所有 section 一致。暖灰站点不在第 7 节突然出现蓝色 CTA。玫红强调站点不在 footer 出现青色状态徽章。

## 6. 暗色模式规则

### 6.1 基础要求

- 消费者面向页面强制设计两种模式
- 禁止只输出 light-only 或 dark-only
- 用 Tailwind `dark:` variant 或 CSS variables，一个项目选一种策略
- 尊重 `prefers-color-scheme: dark`，默认系统偏好

### 6.2 对比度要求

- 两种模式都通过 WCAG AA（正文 AAA）
- 不用纯 `#000000` 和纯 `#ffffff`，用 off-black 和 off-white
- 暗色模式背景推荐：`#0a0a0a`、`#111111`、`#18181b`（zinc-900/950）
- 暗色模式文字推荐：`#fafafa`、`#e4e4e7`、`#d4d4d8`

### 6.3 暗色模式转换规则

| 元素 | Light | Dark |
|------|-------|------|
| 背景 | `bg-white` / `bg-gray-50` | `bg-zinc-950` / `bg-zinc-900` |
| 卡片 | `bg-white` + `border-gray-200` | `bg-zinc-900` + `border-zinc-800` |
| 正文 | `text-gray-600` | `text-zinc-400` |
| 标题 | `text-gray-900` | `text-zinc-100` |
| 强调色 | 保持品牌色 | 保持品牌色（微调亮度） |

## 7. 色彩阴影规则

### 7.1 阴影调色

阴影色调匹配背景色，不用纯黑 drop shadow：

```css
/* ❌ 禁止 */
box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);

/* ✅ 推荐 - 白色背景 */
box-shadow: 0 4px 6px rgba(0, 0, 0, 0.05), 0 10px 15px rgba(0, 0, 0, 0.03);

/* ✅ 推荐 - 有色背景（如蓝色背景） */
box-shadow: 0 4px 6px rgba(37, 99, 235, 0.1), 0 10px 15px rgba(37, 99, 235, 0.05);
```

### 7.2 阴影规则

| 规则 | 说明 |
|------|------|
| 有色阴影 | 阴影色调匹配背景色或强调色 |
| 扩散阴影 | 用 `shadow-[0_20px_40px_-15px_rgba(0,0,0,0.05)]` 替代硬阴影 |
| 内阴影 | 玻璃态用 `shadow-[inset_0_1px_0_rgba(255,255,255,0.1)]` |
| 纯黑禁止 | 不用 `rgba(0,0,0,0.3)` 级别的深阴影 |

## 8. AI 紫蓝渐变指纹识别

### 8.1 检测清单

以下模式是 AI 紫蓝渐变的标准指纹。检测到任意一个 → 标记为 AI 痕迹并修复：

- [ ] Hero 背景：深紫到亮蓝径向渐变
- [ ] 按钮：紫色发光 + 白色文字
- [ ] 装饰线条：霓虹紫 glow
- [ ] 标题：紫色渐变文字
- [ ] 卡片：紫色模糊圆点装饰
- [ ] 导航：紫色高亮
- [ ] 图标：紫色渐变填充

### 8.2 修复方案

| AI 痕迹 | 修复为 |
|---------|--------|
| 紫蓝渐变 Hero | 中性基色 + 单一强调色 Hero |
| 紫色发光按钮 | 品牌色实心按钮 + hover 微交互 |
| 紫色渐变标题 | 纯色标题 + 字重控制 |
| 紫色装饰圆点 | 品牌色装饰元素或移除 |

## 10. 色板纪律总结

| 规则 | 优先级 | 不可协商？ |
|------|--------|-----------|
| 单一强调色 | 最高 | 是（除非品牌多色体系） |
| 单一灰色系 | 最高 | 是 |
| 不混冷暖灰 | 最高 | 是 |
| 不用纯黑/纯白 | 高 | 是（印刷除外） |
| Premium Consumer 色板禁令 | 高 | 是（除非品牌指定） |
| The Lila Ban | 高 | 是（除非品牌指定紫色） |
| Em-Dash 禁令 | 高 | 是（零例外） |
| Color Consistency Lock | 最高 | 是 |
| 阴影调色 | 中 | 否 |
| 暗色模式 | 中 | 消费者页面强制 |
